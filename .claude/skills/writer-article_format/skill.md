# AIdiver 記事フォーマット執筆スキル

このスキルを呼び出したら、以下のステップで記事を執筆する。

---

## Step 1: ヒアリング（素材が揃っていない場合）

以下を確認してから執筆を開始する。

| 項目 | 確認内容 |
|------|---------|
| 記事種別 | インタビュー記事 / 解説記事 / 事例記事 |
| テーマ | 何について書くか |
| ターゲット読者 | 誰に読んでほしいか |
| 素材 | 文字起こし・メモ・参考資料の有無 |
| 連載名 | 連載に属するか（任意） |
| 著者・撮影者 | クレジット情報 |
| ページ分割 | 何ページ構成にするか（目安：3,000字/ページ） |

---

## Step 2: メタ情報の設定

記事冒頭に以下のメタブロックを出力する。

```
【記事メタ情報】
タイトル      :
サブタイトル  :（記事の核心を1文で）
連載名        :（任意）
カテゴリタグ  : AI / 生成AI / 組織 / DX / 人材 など該当するものを選択
著者          :
公開日        :
ページ構成    : 全〇ページ
```

---

## Step 3: 記事本文の執筆

### 全体構成テンプレート

```
# タイトル
## サブタイトル
（ページ X/Y）

---

【リード文】
約150字。記事の背景・読む意義・読後に得られるものを示す。

---

## H2大見出し（記事のブロックごとに設置）

### H3小見出し（論点ごとに設置）

本文...

### H3小見出し

本文...

---

## H2大見出し

...

---

【まとめ】（最終ページのみ）
3〜5行で記事全体の要点を整理する。

【次のアクション】
AIdiverへの入会・関連コンテンツへの誘導を1文で添える。
```

---

### 記事種別ごとのルール

#### インタビュー記事

- 質問は `──質問文` （エムダッシュ＋質問）で表記する
- 回答は `**話者名：**` で話者を明示してから本文
- 質問と回答のセットを1ブロックとして、H3見出しでテーマを区切る
- 編集部の補足・解説は `（編集注：〇〇）` で括弧書きにする

```
## H2：このブロックのテーマ

### H3：この問いのポイント

──質問文をここに書く

**野口：** 回答本文...（ですます調）

──次の質問

**野口：** 回答本文...
```

#### 解説記事

- PREP法（結論→理由→事例→まとめ）を各H2ブロックで使う
- H2は3〜5個を目安にする
- 各H2の末尾に「次の見出しへのブリッジ文」を1文入れる

---

### 文体・トーンのルール

| ルール | 詳細 |
|--------|------|
| 基本文体 | ですます調 |
| 引用・発言 | である調でそのまま再現 |
| 1文の長さ | 60字以内を目安 |
| 対比表現 | 積極的に使う（「かつては〇〇。しかし今は△△」） |
| 専門用語 | 初出時は括弧書きで平易な言葉を補足 |
| 根拠 | 数値・事例・出典を必ず添える。`（※出典名）` で明記 |
| 脚注 | 補足情報は `（※〇〇）` で本文中に括弧挿入 |

---

### 禁止表現・NG事項

- 「AIで何でもできる」などの根拠のない断言
- 読者を不安にさせるだけで解決策のない展開
- 出典のない数値・統計の引用
- `[要確認]` タグを消さずに完成稿として渡すこと

---

## Step 4: ページ分割

- 1ページあたり約2,500〜3,000字を目安にする
- 各ページの末尾に「次のページ：次ページの見出し名」を記載する
- 最終ページにまとめとCTAを入れる

```
---
次のページ：〇〇（次ページのH2見出し）→
```

---

## Step 5: 完成チェック＆ハンドオフ

執筆完了後、以下を自己チェックしてから editor に渡す。

- [ ] リード文が約150字か
- [ ] H2・H3の見出し階層が正しいか
- [ ] ですます調が統一されているか
- [ ] 数値・固有名詞に `[要確認]` が付いているか（未確認のものに限る）
- [ ] 最終ページにまとめとCTAがあるか
- [ ] メタ情報ブロックが冒頭にあるか

チェック完了後、`workplace/projects/<project-name>/` に保存し、editor にレビューを依頼する。

---

## 出力フォルダ

- 執筆中・初稿: `workplace/projects/<project-name>/`
- 完成稿（editor通過後）: `data/documents/`
