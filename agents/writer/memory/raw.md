# Raw Memory

## 2026-04-26 | 野口竜司さん紹介記事の執筆

### 作業概要
- スキル `/writer-article_format` を使い、AI専門家・野口竜司氏の人物紹介記事を執筆
- WebSearch・WebFetchでリサーチ → 2ページ構成（約5,000字）の解説記事を生成
- editorに校閲依頼 → 差し戻し2点 → 追加事実確認 → 解消・完了

### 執筆コンテンツ
- タイトル：「文系でもAIは使いこなせる」——野口竜司が切り拓く、AIトランスフォーメーションの現在地
- 完成稿：`data/documents/2026-04-26_noguchi-ryuji-article.md`

### 確認済み事実（次回の参考情報）
- 野口竜司 = AIX partner代表 / 元ELYZA CMO / 元ZOZO NEXT取締役CAIO
- 著書：『文系AI人材になる』2019年 / 『ChatGPT時代の文系AI人材になる』2023年（共に東洋経済新報社）
- Z AIアカデミア 発起人 兼 幹事（EnterpriseZine・ZOZO公式で確認）
- グロースX AI戦略アドバイザー（プレスリリースで確認）

### 苦労した点・工夫した点
- WebFetchが画像・JavaScriptページを取得できない場合があり、複数ソースを組み合わせてリサーチした
- editorへの引き継ぎ時に「確認済みリスト」を付けたが、リストに含めていない肩書きが2点あり差し戻しになった

[INSIGHT] 人物紹介記事では、すべての肩書きを「出典URL付き確認済みリスト」にまとめてからeditorに渡すと差し戻しを防げる。

[INSIGHT] 役職名の一次情報として最も信頼性が高いのは「公式プレスリリース（PRTIMES等）」と「所属団体の公式ページ」。メディア記事・Amazonは補助情報として扱う。

[INSIGHT] 人物の愛称・SNSアカウントをリード文に入れると、読者との距離感が縮まり親しみやすくなる（「のぐりゅう」の効果）。
